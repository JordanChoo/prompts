# PRD Review

Reviews an initial human created PRD to identify opportunities for improvement, detail, and refinment. Primarily for a web version of an LLM with a PRD as an .md file 

Can also be adapted to CLI versions with slight changes

```
Ruthlessly, unapologetically, and with extreme detail review the attached MD file to identify any bugs, edge cases, lack of clarity, and potential issues that may arise with it

Think that if you had 20/20 vision 10 years from now know what you know what would you do differently to make sure it is future proofed. The roadmap file will eventually be passed to a development team and they should know EXACTLY what needs to be done and how without any other context

Before providing me with recommendations, ask me questions to clarify exactly what I want built and my expectations so your review is as clear and thorough as possible
```
